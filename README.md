# javascript-lotto-precourse

# 로또를 만들자

### 기능 요구사항 명세

```
입력
- 1. 구매 금액 입력
     1000원 단위로 입력 받고 나눠 떨어지지 않는경우는 예외처리한다
- 2. 당첨 번호를 입력 받는다
     당첨번호는 총 6개
     쉼표를 기준으로 구분한다
     당첨번호는 증복되어선 안된다
- 3. 보너스 번호를 입력 받는다
     당첨번호와 증복되어선 안된다

* 사용자가 잘못입력시 [ERROR]로 시작하는 메시지를 출력하고 [ERROR] 발생 후 다음 해당지점부터 다시 입력을 받는다
```

```
예시
- 1. 14000
- 2. 1,2,3,4,5,6
- 3. 7
```

```
출력
- 1. 발행 로또 개수와 번호를 출력
    - 로또 번호는 오름차순으로 정렬해야한다

- 2. 당첨 내역을 출력합니다
     - 3 ~ 6 순서

- 3. 수익률을 출력
     - 소수점 둘째 자리에서 반올림 한다


* 로또 당첨 가격
1등: 6개 번호 일치 / 2,000,000,000원
2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
3등: 5개 번호 일치 / 1,500,000원
4등: 4개 번호 일치 / 50,000원
5등: 3개 번호 일치 / 5,000원
```

```
1.예시
    8개를 구매했습니다.
    [8, 21, 23, 41, 42, 43]
    [3, 5, 11, 16, 32, 38]
    [7, 11, 16, 35, 36, 44]
    [1, 8, 11, 31, 41, 42]
    [13, 14, 16, 38, 42, 45]
    [7, 11, 30, 40, 42, 43]
    [2, 13, 22, 32, 38, 45]
    [1, 3, 5, 14, 22, 45]
2.예시
    3개 일치 (5,000원) - 1개
    4개 일치 (50,000원) - 0개
    5개 일치 (1,500,000원) - 0개
    5개 일치, 보너스 볼 일치 (30,000,000원) - 0개
    6개 일치 (2,000,000,000원) - 0개
3. 예시
    총 수익률은 62.5%입니다.
4. 예시
    [ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.
```

```
프로그래밍 요구 사항
1. indent depth 3 이상 X
2. 3항 연산자 쓰지 말것
3. 함수가 한가지 일만 하도록 최대한 작게 만들기
4. jest를 이용한 단위테스트 작성
5. 함수의 길이가 15자 이내
6. else 지양
```

```
Lotto 클래스
1. 로또 클래스 이용할것
2. 필드 (인스턴스 변수) 추가 불가능
3. numbers의 # 변경 불가
4. Lotto 패키지 변경 가능

```
